# OctoAcme Project Management Docs

This README is the entry point for OctoAcme's project management documentation set. It improves discoverability by giving contributors a concise overview of the process and quick access to each guide in `docs/`.

## Summary

OctoAcme follows a lightweight, iterative lifecycle that moves work through initiation, planning, execution, release, and retrospective stages. Each stage emphasizes clear ownership, measurable outcomes, and small, testable increments so teams can align on goals, prioritize scope, deliver against milestones, and capture learnings for continuous improvement.

Execution is supported by a regular team rhythm that includes daily standups, weekly delivery syncs, sprint or milestone demos, and recurring stakeholder updates. Work is managed through a visible project board and small pull requests that link issues, include acceptance criteria, and pass CI checks before review so progress, blockers, risks, and dependencies stay visible.

Roles and communication practices are intentionally explicit across the docs. Project Managers coordinate schedules, risks, dependencies, and reporting; Product Managers define outcomes and prioritize the backlog; Developers implement, test, and help surface technical risks; and QA or testing stakeholders validate acceptance criteria and release readiness. Quality assurance is reinforced through unit, integration, and smoke testing, manual QA when needed, security scanning, release readiness checks, and retrospectives that convert lessons learned into tracked action items.
